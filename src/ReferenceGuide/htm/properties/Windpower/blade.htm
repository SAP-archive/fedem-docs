Blade

The Blade object is a sub-assembly used to represent a turbine blade in a wind turbine model.

The Blade sub-assembly contains objects of type Beam and Triad in its model tree list, representing the individual segments of the blade. You must use the Blade Definition dialog box to create or modify blades. You should not try to add or remove blade segments manually.

The figure above illustrates the Blade properties, which consist of the following items:

Name:Description:
Pitch controlSpecifies a value or function for the pitch angle of the blade.
Fixed pitchSpecifies that the pitch is fixed (no change).
MassDisplays the total mass of the blade.
LengthDisplays the total length of the blade.
Add ice layerAdds an ice layer on the blade.
ThicknessSpecifies the thickness of the ice layer.
Mass proportionalSpecifies mass proportional structural damping.
Stiffness proportionalSpecifies stiffness proportional structural damping.
Stiffness scaleSpecifies stiffness scale.
Mass scaleSpecifies mass scale.
Visualize 3DEnables or disables 3D visualization of the blade geometry.

The setting of the Pitch control field is also reflected in the Rz-values of the corresponding revolute joint in the root of the blade, which can be edited if more advanced pitch modeling is needed, e.g., adding flexibility and damping in the pitching motors, etc.

Adding an ice layer results in extra non-structural mass on each blade segment. The ice mass is estimated based on a mass density of 916.7 kg/m3 and an ellipsoidal cross section, using the specified Chord length and Thickness ratio parameters of the Blade Definition to deduce the length of the principal axes of the ellipse.

Editing any of the Structural Damping and Scaling of Dynamic Properties fields, the values entered will apply to all the underlying Beam elements of the blade, which can be verified by selecting any of the beam elements. But, if such a field is edited for one beam element such that the beams making up a blade no longer have identical properties, the corresponding field in the Blade property panel will be blank (no value displayed).

Note: The Visualize 3D toggle has a tri-state value. In addition to the usual ON and OFF states, it can also have a neither-ON-nor-OFF state, in which the corresponding ON/OFF settings on each underlying beam element are applied instead. This is usually the default setting for such tri-state toggles.
