Beam

The Beam object is used to to represent a two-noded beam finite element in a FEDEM model. The Beam is connected to a Triad at each end of the element. However, an arbitrary number of beam elements may be connected in one Triad, such that it is possible to represent both a curved (or straight) beamstring and 3D spaceframe structures.

The figure above illustrates the Beam properties, which consist of the following items:

Name:Description:
Cross-sectionThis pull-down list enables the selection of the Beam cross section to be used for this beam element.
Mass and LengthDisplays the length and the total mass of the beam element. The length is computed from the coordinates of the end triads, and the mass is computed from the length, and the cross section area and mass density, assigned via the cross section object. These fields are updated automatically if any of the referred property objects are changed.
Visualize 3DThis toggle enables a 3D visualization of the beam, using the cross section parameters to render its actual geometry. The default visualization is a straigth line connecting the two triads, with coloring depending on the referred cross section object. The Start and Stop fields are used to specify a sector of partial 3D visualization. They define the start and stop angles (in degrees), relative to the local Y-axis of the cross section, of the sector that is to be rendered in 3D.
Structural DampingAllows you to change the values of the mass- and stiffness-proportional damping for the Beam. These fields are equivalent to similar fields in the Part tab of the Part property panel.
Scaling of Dynamic PropertiesAllows you to scale stiffness and mass of each individual Beam in the dynamics simulation. These fields are equivalent to similar fields in the Part tab of the Part property panel.
Local Z-axis definitionThese three fields allow you to specify a vector defining the Z-axis of the local coordinate system of the beam element. This vector is defined in the coordinate system of the parent Subassembly of the Beam, or in the global coordinate system in case of no parent assembly. The local Y-axis is then defined as the cross product between the Z-axis and the local X-axis, the latter being the unit vector from end 1 towards end 2 of the element.
NOTE: If the specified vector is parallel to the local X-axis, or if all fields are zero, a globalized coordinate system is used instead in which the local Z-axis is chosen to be as close as possible to the global Z-axis.

Important: Be careful when modifying the Beam properties when multiple beams are selected. FEDEM will assign the values in the property panel to all the selected items. You should therefore not set Visualize 3D on multiple beams by multiselecting them, since it will then change all properties of all selected beams.
