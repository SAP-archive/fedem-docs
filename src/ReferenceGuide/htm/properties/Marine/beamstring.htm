Beamstring

The Beamstring object is a sub-assembly used to represent long beam structures, such as risers, conductors, umbilicals, etc.

The Beamstring sub-assembly normally contains sets of Beams and Triads in its model tree list. In addition, it contains some Beam cross sections and Materials used by the Beam elements.

Please see the chapter on "Marine Modeling" in the FEDEM User’s Guide for more information on how to create beamstring assemblies.

Name:Description:
Subassembly model fileSpecifies a model file for the beamstring. A separate model file may be assigned to the beamstring assembly by using the Browse button.
Is internal to:Currently unused.
MassDisplays the total mass of the beamstring (excluding additional mass due to marine growth and/or internal liquid).
LengthDisplays the total length of the beamstring.
Visualize 3DEnables or disables 3D visualization of the Beam elements, using the cross section parameters to render its actual geometry. If disabled, each beam is visualized as a straigth line connecting the two triads, with coloring depending on the referred Beam cross section object.
Start, StopSpecify start and stop angles (in degrees, relative to the local Y-axis of the beam cross section) for a partial 3D visualization of the beams.
Internal LiquidEnables or disables the calculation of additional mass due to internal liquid (mud) in the beamstring, based on the specified Mud density and Mud level. See the chapter on "Marine Modeling" in the FEDEM User’s Guide for details on this additional mass calculation.
Mass proportionalSpecifies mass proportional structural damping.
Stiffness proportionalSpecifies stiffness proportional structural damping.
Stiffness scaleSpecifies stiffness scale.
Mass scaleSpecifies mass scale.
PositionSpecifies the position of the beamstring assembly.
OrientationSpecifies the orientation of the beamstring assembly.

If a separate model file is assigned to a beamstring assembly, all beams, triads and properties of that beamstring are stored in the specified file when the model is saved. It is then possible to reuse this beamstring in another model by importing it into an existing model. If the Subassembly model file field is left empty, the elements of the beamstring are stored in the main model file together with the rest of the model.

Editing any of the Structural Damping and Scaling of Dynamic Properties fields, the values entered will apply to all the underlying Beam elements of the beamstring, which can be verified by selecting any of the beam elements. But, if such a field is edited for one beam element such that the beams making up a beamstring no longer have identical properties, the corresponding field in the Beamstring property panel will be blank (no value displayed).

Note: The Visualize 3D toggle has a tri-state value. In addition to the usual ON and OFF states, it can also have a neither-ON-nor-OFF state, in which the corresponding ON/OFF settings on each underlying beam element are applied instead. This is usually the default setting for such tri-state toggles.
