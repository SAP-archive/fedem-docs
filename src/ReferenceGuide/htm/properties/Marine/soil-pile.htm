Soil Pile

The Soil Pile object is a sub-assembly used to represent a hollow or solid pile that has been placed in the marine soil.

The Soil Pile sub-assembly normally contains the same sets of objects in its model tree list as a Beamstring, (Beam, Triads, Beam cross sections, and Materials). In addition, it contains a set of Free Joints with corresponding Spring characteristic objects, representing the soil behaviour in the model.

Please see the chapter on "Marine Modeling" in the FEDEM User’s Guide for more information on how to create soil pile assemblies.

The figure above illustrates the Soil Pile properties, which consist of the following items:

Name:Description:
Subassembly model fileSpecifies a model file for the soil pile. A separate model file may be assigned to the soil pile assembly by using the Browse button.
MassDisplays the total mass of the soil pile (excluding additional mass due to internal soil).
LengthDisplays the total length of the soil pile.
Visualize 3DEnables or disables 3D visualization of the Beam elements, using the cross section parameters to render its actual geometry. If disabled, each beam is visualized as a straigth line connecting the two triads, with coloring depending on the referred Beam cross section object.
Start, StopSpecify start and stop angles (in degrees, relative to the local Y-axis of the beam cross section) for a partial 3D visualization of the beams.
Internal SoilEnables or disables the calculation of additional mass due to internal soil in the pile, based on the specified soil density.
Mass proportionalSpecifies mass proportional structural damping.
Stiffness proportionalSpecifies stiffness proportional structural damping.
Stiffness scaleSpecifies stiffness scale.
Mass scaleSpecifies mass scale.
PositionSpecifies the position of the soil pile assembly.
OrientationSpecifies the orientation of the soil pile assembly.

If a separate model file is assigned to a soil pile assembly, all beams, triads and properties of that soil pile are stored in the specified file when the model is saved. It is then possible to reuse this soil pile in another model by importing it into an existing model. If the Subassembly model file field is left empty, the elements of the soil pile are stored in the main model file together with the rest of the model.

Editing any of the Structural Damping and Scaling of Dynamic Properties fields, the values entered will apply to all the underlying Beam elements of the soil pile, which can be verified by selecting any of the beam elements. But, if such a field is edited for one beam element such that the beams making up a soil pile no longer have identical properties, the corresponding field in the Soil Pile property panel will be blank (no value displayed).

Note: The Visualize 3D toggle has a tri-state value. In addition to the usual ON and OFF states, it can also have a neither-ON-nor-OFF state, in which the corresponding ON/OFF settings on each underlying beam element are applied instead. This is usually the default setting for such tri-state toggles.
