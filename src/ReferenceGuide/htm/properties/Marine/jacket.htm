Jacket

The Jacket object is a sub-assembly used to represent a marine spaceframe structure.

The Jacket sub-assembly normally contains sets of Beams and Triads in its model tree list. In addition, it contains some Beam cross sections and Materials used by the Beam elements.

Please see the chapter on "Marine Modeling" in the FEDEM User’s Guide for more information on how to create spaceframes.

The figure above illustrates the Jacket properties, which consist of the folliwing items:

Name:Description:
Subassembly model fileSpecifies a model file for the spaceframe. A separate model file may be assigned to the jacket assembly by using the Browse button.
MassDisplays the total mass of the spaceframe structure (excluding additional mass due to marine growth and/or internal liquid).
Center of GravityDisplays the location of the spaceframe's center of gravity, in either Global coordinates or in Local coordinates with respect to the origin of the jacket assembly.
Visualize 3DEnables or disables 3D visualization of the beams, using the cross section parameters to render its actual geometry. If disabled, each beam is visualized as a straigth line connecting the two triads, with coloring depending on the referred Beam cross section object.
Mass proportionalSpecifies mass proportional structural damping.
Stiffness proportionalSpecifies stiffness proportional structural damping.
Stiffness scaleSpecifies stiffness scale.
Mass scaleSpecifies mass scale.
PositionSpecifies the position of the jacket assembly.
OrientationSpecifies the orientation of the jacket assembly.

If a separate model file is assigned to a jacket assembly, all beams, triads and properties of that spaceframe are stored in the specified file when the model is saved. It is then possible to reuse this spaceframe in another model by importing it into an existing model. If the Subassembly model file field is left empty, the elements of the spaceframe are stored in the main model file together with the rest of the model.

Editing any of the Structural Damping and Scaling of Dynamic Properties fields, the values entered will apply to all the underlying Beam elements of the spaceframe, which can be verified by selecting any of the beam elements. But, if such a field is edited for one beam element such that the beams making up a spaceframe no longer have identical properties, the corresponding field in the Jacket property panel will be blank (no value displayed).

Note: The Visualize 3D toggle has a tri-state value. In addition to the usual ON and OFF states, it can also have a neither-ON-nor-OFF state, in which the corresponding ON/OFF settings on each underlying beam element are applied instead. This is usually the default setting for such tri-state toggles.
