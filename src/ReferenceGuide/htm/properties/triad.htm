Triad

The Triad object is used to define connection points between mechanism elements, such as parts, beams, joints, axial springs and dampers, in a FEDEM model. Triads are also used as attack point for external concentrated loads and point masses. A Triad defines a set of three orthogonal coordinate axes at the connection point, which serve as the reference frame for the 6 degrees of freedoms (DOFs) at this point (three translations Tx, Ty, Tz, and three roations Rx, Ry, Rz).

Origin tab

The figure above illustrates the Origin properties of the Triad, which consist of the following items:

Name:Description:
PositionThis frame displays data for the translational part of the Triad location.
OrientationThis frame displays data for the rotational part of the Triad location.
Reference CSEach part of the Triad location can be displayed and edited in any coordinate system defined in the model. These pull-down lists allow the reference coordinate systems to be selected. They can also be selected by picking in the Modeler or selected from the Objects Browser To do so, you must first click the arrow button next to the pull-down list. Any Triad, Beam or Part can be selected.
Coordinate typeThis pull-down lists control how to display and edit the Position and Orientation of the Triad.

In the Position frame, the Coordinate type options are either Cartesian or Cylindrical coordinates, and with Cylindrical coordinates either X, Y or Z can be chosen as the cylinder axis.

In the Orientation frame, the Coordinate type options are as follows:

  • EulerZYX, as X, Y, Z - Angles (in degrees about) the X-, Y- and Z-axes. This means a rotation about the Z-axis of the reference CS performed first, next about the Y-axis, and finally the X-axis.
  • Point on Z-axis, and in the XZ plane - Together with the triad position itself, this uniquely defines the coordinate system.
  • Point on X-axis, and in the XY plane - Together with the triad position itself, this uniquely defines the coordinate system.
  • X vector, and XY vector - The specified X-vector is used directly, and the Z-vector is defined from the cross product between the two vectors.

Triad DOF tabs

These tabs display detailed properties related to each of the six DOFs (Tx-Rz) associated with the Triad. The available options here depend on the selected Constraint type.

The figure above illustrates the Tx-Rz properties of the Triad when the Constraint type is Free, and consist of:

Name:Description:
Load magnitudeYou can apply a Load in this triad DOF. This will be a torque or a force depending on whether this is a translational or a rotational DOF. The load is applied in the defined System direction of the Triad.
Apply in frequency domainIf a Function is specified in the Load magnitude field, this toggle will enable the load to be applied in the frequency domain instead of the time domain, if the Frequency response analysis toggle is enabled in the Solver setup dialog.
Initial velocityYou may specify a velocity that should be applied as initial condition in this DOF, i.e., the actual velocity at the start time of the dynamics simulation.
Additional BCWhen this toggle is enabled, the movement in this triad DOF is fixed during the initial equilibrium analysis and optionally also in the eigenmode analysis. (See the Eigenmode tab, and the FEDEM Theory Guide, Section 7.8 "Quasistatic equilibrium".)

When the Constraint type is Fixed, no further property items are available as illustrated in the figure above.

The figure above illustrates the Tx-Rz properties of the Triad when the Constraint type is Prescribed, and consist of:

Name:Description:
Motion magnitudeYou can prescribe a constant deflection, velocity or acceleration by typing the desired value into this field, or a time-dependent evolution of the motion during the simulation by selecting a function.
Prescribed quantityYou may choose whether you want to prescribe the Deflection, the Velocity or the Acceleration for the triad DOF.
Apply in frequency domainIf a Function is specified in the Motion magnitude field, this toggle will enable the prescribed motion to be applied in the frequency domain instead of the time domain, if the Frequency response analysis toggle is enabled in the Solver setup dialog.
Initial velocityYou may specify a velocity that should be applied as initial condition in this triad DOF, i.e., the actual velocity at the start time of the dynamics simulation.

Caution: When using a prescribed non-constant motion in triad DOF, it is the analysts responsibility to also specify an initial velocity that complies with the specified motion. Failure to do so may result it unwanted ficticious transcients in the beginning of the simulation.

You may plot the reaction force associated with a fixed or prescribed triad DOF by selecting the "Force" or "Moment" item under the Triad node in question form the RDB selector (see "Selecting RDB results" in FEDEM User's Guide, "Curve properties").

Results tab

The figure above illustrates the Results properties of the triad. It contains toggles for activating output of certain solution variables related to the triad, such that the quantities are available for curve plotting, etc.

In the Global frame, the following toggles are available:

Name:Description:
VelocityOutput velocities in global coordinates for the triad when this toggle is enabled, no output when disabled.
AccelerationOutput accelerations in global coordinates for the triad when this toggle is enabled, no output when disabled.
ForceOutput forces in global coordinates for the triad when this toggle is enabled, no output when disabled.
DeformationOutput deformational displacements for the triad when this toggle is enabled, no output when disabled.

In the Local frame, the following toggles are available:

Name:Description:
VelocityOutput velocities in the local coordinate system of the triad when this toggle is enabled, no output when disabled.
AccelerationOutput accelerations in the local coordinate system of the triad when this toggle is enabled, no output when disabled.
ForceOutput forces in the local coordinate system of the triad when this toggle is enabled, no output when disabled.
