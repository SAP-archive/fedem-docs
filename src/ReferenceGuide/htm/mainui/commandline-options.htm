Command-line Options

Each of the FEDEM programs may be run manually from a console window or using the Run... option from the Start menu in Windows. To facilitate such batch-execution of the programs, the complete list of command-line options for each solver module is given in Appendix C in the FEDEM User's Guide. Below, the options for the FEDEM user interface (i.e., Fedem.exe) are shown.

Command-line option:Description:Default value:
-checkRDBinterval Time [ms] between each RDB check/update during solve 500
-console Enable console window - (false)
-debug Run in debug mode - (false)
-events Simulation event definition file
-exportAnimations Auto-export animations to VTF on batch solve - (false)
-exportCurves Auto-export curves on batch solve. Specify folder to export curve files to.
-f Model file to open untitled.fmm
-help Display this help and exit - (false)
-logFile Write all Output List contents to log-file. Log-file name: (modelFilePrefix).log + (true)
-noFEData Load model file without FE-Models and FE-Results info. Use together with -f. - (false)
-plotElements Enable plotting of element results - (false)
-plotNodes Enable plotting of nodal results - (false)
-prepareBatch Prepare for batch execution. This option can have the following arguments:
all = all solvers
reducer = reduction of all parts
dynamics = dynamics solver
stress = stress recovery for in parts
modes = mode shape recovery in all parts
straingage = strain gage recovery in all parts
straincoat = strain coat recovery in all parts
-purgeOnSave Purge inactive mechanism objects on Save - (false)
-solve Start given solver(s) in batch mode. This option can have the following arguments:
all = all solvers
events = all solvers on all events
reducer = reduction of all parts
dynamics = dynamics solver
stress = stress recovery for in parts
modes = mode shape recovery in all parts
straingage = strain gage recovery in all parts
straincoat = strain coat recovery in all parts
-version Display program version and exit - (false)
-timerange Time specification for batch stress recovery.
Format: [startTime,stopTime(,timeInc)]. timeInc is optional.
